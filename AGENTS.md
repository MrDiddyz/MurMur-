# Agent Rules (Global)

These instructions apply to the entire repository unless a deeper `AGENTS.md` overrides them.

## Workflow
- Keep changes atomic and easy to review.
- Prefer small, focused commits.
- Run relevant checks before committing.

## Documentation
- Keep `codex/repo_map.md`, `codex/runbook.md`, and `codex/quality_gate.md` aligned with code changes.
- Add or update ADRs under `docs/decisions/` for architectural choices.

## Pull Requests
- Use `.github/pull_request_template.md` when preparing PR descriptions.
- Include summary, validation steps, and risk notes.
